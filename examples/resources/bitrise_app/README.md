# Bitrise App Resource Example

This example demonstrates how to register a Bitrise application using Terraform.

## Usage

1. Set your Bitrise API token:
   ```bash
   export TF_VAR_bitrise_token="your-bitrise-api-token"
   ```

2. Update the configuration:
   - Replace `myorg` with your GitHub organization or username
   - Replace `myrepo` with your repository name
   - Replace `my-bitrise-org` with your Bitrise organization slug

3. Initialize and apply:
   ```bash
   terraform init
   terraform plan
   terraform apply
   ```

## What This Example Does

This example creates two Bitrise applications:

1. **Private App** (`my_app`): A private GitHub repository connected to Bitrise
2. **Public App** (`public_app`): A public GitHub repository configured for open-source projects

## Outputs

After applying, the following outputs are available:

- `app_slug`: The unique slug identifier for the created app (used for other Bitrise resources)
- `app_id`: The ID of the app (same as app_slug)

## Next Steps

After creating an app, you typically want to:

1. **Configure SSH Keys**: Use `bitrise_app_ssh` to add SSH keys for repository access
2. **Complete Setup**: Use `bitrise_app_finish` to configure project type, stack, and build configuration
3. **Add Secrets**: Use `bitrise_app_secret` to add environment variables and secrets
4. **Configure Roles**: Use `bitrise_app_roles` to manage team access

## Supported Git Providers

- `github` - GitHub
- `gitlab` - GitLab
- `bitbucket` - Bitbucket
- Others as supported by Bitrise

## Notes

- The `app_slug` is automatically generated by Bitrise and cannot be customized
- Changes to repository URL or organization will force recreation of the app
- Make sure your Bitrise account has access to the specified repository
